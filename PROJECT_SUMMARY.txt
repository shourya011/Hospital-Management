📋 DATABASE SCHEMA

TABLE: patients (5 seed records)
  ┌─ patient_id (INT, PK, AUTO_INCREMENT)
  ├─ first_name, last_name (VARCHAR)
  ├─ date_of_birth (DATE)
  ├─ gender (ENUM: Male, Female, Other)
  ├─ blood_group (VARCHAR: A+, A-, B+, B-, O+, O-, AB+, AB-)
  ├─ phone (VARCHAR, UNIQUE)
  ├─ email (VARCHAR)
  ├─ address, city, state, pincode (VARCHAR)
  └─ registered_on (TIMESTAMP)

TABLE: doctors (5 seed records)
  ┌─ doctor_id (INT, PK, AUTO_INCREMENT)
  ├─ name (VARCHAR): Dr. Rajesh Kumar, Dr. Priya Sharma, etc.
  ├─ specialization (VARCHAR): Cardiologist, Neurologist, etc.
  ├─ phone (VARCHAR)
  └─ available_days (VARCHAR)

TABLE: appointments (5 seed records + auto-generated)
  ┌─ appointment_id (INT, PK, AUTO_INCREMENT)
  ├─ patient_id (INT, FK → patients)
  ├─ doctor_id (INT, FK → doctors)
  ├─ appointment_date (DATE)
  ├─ appointment_time (TIME)
  ├─ reason (TEXT)
  ├─ status (ENUM: Scheduled, Completed, Cancelled)
  ├─ token_number (INT, auto-generated)
  └─ created_at (TIMESTAMP)
